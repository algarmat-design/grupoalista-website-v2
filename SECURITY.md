# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please send an email to ventas@grupoalista.com. All security vulnerabilities will be promptly addressed.

Please do not publicly disclose the issue until it has been addressed by the team.

## Security Best Practices

This project follows these security guidelines:

- No hardcoded secrets or credentials
- No use of `eval()` or `innerHTML` with user input
- All external links use `rel="noopener noreferrer"`
- Content Security Policy headers recommended for deployment
- Regular dependency updates to patch vulnerabilities
